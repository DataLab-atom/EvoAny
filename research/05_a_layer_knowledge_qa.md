# 能力分层架构

---

# A 层：知识检索与问答

> 统一的文献能力层——本地优先、外部补充、自动入库

## A1 在线文献检索

针对给定内容，从互联网检索相关文献并返回。

## A2 本地文献检索

从本地现存的文献库中检索匹配内容并返回答案。

## A3 代码问答

基于 baseline 代码、新代码及改动历史，回答传入的问题。

## A5 代码差异分析

对演化前后的代码进行结构化 diff 分析，为 C 层迭代提供动态更新的改动视图。

- **输入**：baseline 代码、演化后代码（或 git commit 范围）
- **流程**：
  1. 提取 diff 并按文件 / 函数粒度分组
  2. 对每组改动进行语义标注（新增逻辑、参数变更、结构重组…）
  3. 输出改动聚类——哪些改动可能服务于同一目的
- **输出**：结构化改动报告（改动列表 + 语义标签 + 聚类建议）
- **与 A3 的区别**：A3 是被动问答（给定问题回答），A5 是主动分析（给定代码输出改动全景）

## A4 文献问答（对外接口）

- **输入**：一个需求/问题
- **流程**：
  1. 优先调用 **A2**（本地文献库）获取相关文献
  2. 数量不足时调用 **A1**（在线检索）补充
  3. 在线检索到的论文**自动写回本地文献库**
- **输出**：基于检索到的文献展开回答，附带 BibTeX 标准引用格式

# B 层：实验执行

> 两大职责：**可视化分析**（画图 → 找亮点）与**基准补充**（刷 benchmark → 补实验）。
> 两条线都与 A 层保持交互，确保结果正确、图表规范。

---

## B-I 可视化分析（画图找亮点）

> 根据上层传入的期望结论，生成精致的分析图表，验证实验结果是否与期望一致，挖掘亮点。

### B1 期望驱动出图

根据传入的期望（如"我们的损失变化比 baseline 更稳定"），选择合适的图表类型并生成初版图。

- **输入**：期望结论、实验数据（来自已有实验结果）
- **流程**：
  1. 解析期望结论，确定需要对比的指标和维度
  2. 选择图表类型（loss 曲线、柱状图、热力图等）
  3. 生成初版分析图
- **输出**：初版图表 + 数据是否支持期望的判定

### B2 亮点挖掘与一致性分析

对比图表结果与传入期望，判断是否一致；如果一致，进一步挖掘可突出的亮点。

- **输入**：初版图表、期望结论
- **流程**：
  1. 定量对比实验数据与期望的吻合度
  2. 若一致，标注图中最能体现优势的区域/数据点
  3. 若不一致，反馈偏差原因供上层调整
- **输出**：亮点标注报告（哪些数据点/区间最能支撑贡献点）

### B3 图表精修（与 A 层交互）

主体结论确认无误后，与 A 层交互，参考领域内高水平论文的图表风格，对图进行美观性和规范性提升。

- **输入**：确认无误的初版图表
- **流程**：
  1. 调用 **A4** 检索目标期刊/会议的图表规范与高引论文的图表示例
  2. 根据 A 层反馈调整配色、字体、图例、坐标轴标注等
  3. 确保符合投稿目标的格式要求（分辨率、字号、单双栏等）
- **输出**：精修后的出版级图表

---

## B-II 基准补充（刷 benchmark）

> 根据上层传入的补充实验需求，适配新数据集/新设置，跑出性能，返还结果。

### B0 实验设计

根据 C 层传入的假说，设计可验证的实验方案，再交由 B4–B6 执行。

- **输入**：待验证假说（如"性能提升主要来自注意力机制的改动"）
- **流程**：
  1. 解析假说的可测试变量（自变量、因变量、控制变量）
  2. 生成实验方案：消融实验（逐项去除改动）、对照实验（替换为替代方案）、或组合实验
  3. 调用 **A4** 检索领域内同类假说的标准验证方式，确保实验设计符合领域惯例
  4. 输出具体的实验清单，每项包含：改动内容、对比基线、预期结果、判定标准
- **输出**：结构化实验方案（可直接作为 B4 的输入）

### B4 数据集适配

根据需求对现有方法代码进行数据集适配，使其能在新 benchmark 上运行。

- **输入**：补充实验需求（如"需要在 X、Y 数据集上补充 SOTA 对比"）、现有方法代码
- **流程**：
  1. 调用 **A4** 确认目标数据集的标准评测协议（数据划分、评价指标、预处理方式）
  2. 调用 **A3** 分析现有代码的数据加载接口
  3. 编写/修改数据适配代码（dataloader、预处理、评价脚本）
- **输出**：可直接运行的适配代码

### B5 执行与结果收集

在沙箱中运行适配后的代码，收集性能数据。

- **输入**：适配代码、目标数据集
- **流程**：
  1. 沙箱环境中执行训练/推理
  2. 收集各指标的数值结果
  3. 整理为标准对比表格（含 baseline 和 SOTA 方法的公开数据）
- **输出**：性能对比表格

### B6 结果校验（与 A 层交互）

跑完后与 A 层交互，校验结果是否合理、是否有遗漏。

- **输入**：性能对比表格
- **流程**：
  1. 调用 **A4** 检索目标数据集上已发表方法的 SOTA 数值
  2. 对比自身结果与公开 SOTA，判断是否合理（异常高/低需排查）
  3. 调用 **A4** 确认是否还需补充其他常用 benchmark 或消融实验
- **输出**：校验报告（结果是否合理 + 是否需要补充更多实验）

# C 层：研究验证回环

> 核心机制：推导森林。从已知有效的演化结果出发，通过多轮迭代深挖"为什么有效"，
> 让多条推导分支在深层自然交汇，浮现出真正的深层动机和贡献点。
> A 层、B 层是 C 层调用的工具包，D/E 层是 C 层产出的下游消费者。

- **输入**：Evo-anything 的演化结果（代码 + 性能数据，已知有效）
- **输出**（全部存 git）：
  - 推导森林的完整结构（JSON 序列化）
  - 深层动机及其验证（自然语言文本，质量由写作智能体负责）
  - 主贡献（交汇分支）+ 辅助贡献（未交汇分支）
  - 所有实验证据（通过标记过的 git 分支管理，每轮验证对应一个分支）
  - 文献关系图谱（多类型节点：现存问题、论文、观点等；边为节点间的自然关系）

## 推导森林回环

初始状态为粗粒度感知：代码变了，效果好了。对改动的理解不在回环之前预设，
而是在回环中动态形成和修正。

### 每一轮迭代

1. **以当前理解切入代码改动**（调 **A5** 获取结构化 diff）
   - 改动的边界和含义随迭代动态修正
   - 可能合并之前认为独立的改动
   - 可能拆分之前认为统一的改动
   - 森林节点动态增删合并

2. **反推：为什么有效？**
   - 构建 / 扩展推导树节点
   - 从结果反推：到底是解决了什么领域尚未解决的问题，才导致性能大幅提升？

3. **调 A 层查文献**
   - 这个机制在理论上有无解释？（数学 / 物理 / 信息论…）
   - 有没有人已经在研究这个问题？在什么领域？
   - 领域对此的理解到了什么程度？

4. **调 B 层验证**（先调 **B0** 设计实验方案，再交 B4–B6 执行）
   - B0 根据当前假说输出结构化实验方案
   - B4–B6 执行并返回结果
   - 支持 → 当前节点成立，继续深挖
   - 不支持 → 剪枝，换分支

5. **检查交汇**
   - 多条分支的深层根因是否指向同一个问题？
   - 尚未交汇 → 回到步骤 1 继续迭代
   - 出现交汇点 → 进入收束阶段

### 收束阶段

交汇点 Q 就是深层动机——不是预设的，是多条分支深挖后自然浮现的。

1. **验证 Q 的真实性**
   - 调 A 层：Q 在领域中是否被认识到？是已知未解决问题，还是全新洞察？
   - 调 B 层：能否设计实验直接证明 Q 的存在和影响？

2. **贡献定级**
   - 交汇分支 → 主贡献（由深层动机 Q 统一）
   - 未交汇分支 → 降级为辅助贡献

### 核心设计约束

- **无前置差异分析**：对改动的理解在回环中动态形成，不预设、不定死
- **推导森林不是预设的**：边推导边生长，节点动态增删合并
- **动机是推出来的不是编出来的**：多条分支深挖后自然交汇，交汇点即深层动机
- **验证失败是剪枝不是终止**：回到森林换分支继续

# DECB 超级回环

> D 写论文 → E 审核 → 问题按严重程度逐层回踢 → 修复后逐层返回 → E 再审。
> 回环深度取决于问题的严重程度，表面问题在 D 层就地解决，深层问题一路回到 C 甚至 B。

## 回环流程

```
D（写论文）→ E（审核）
                ↓ 审核通过 → 输出终稿
                ↓ 审核不通过 → 按问题分级回踢：
                │
                ├─ 表层问题 → 回 D
                │   格式、语言、LLM痕迹、图表细节、公式符号、引用格式
                │   D 修改后 → 回 E 复审
                │
                ├─ 内容问题 → 回 C
                │   论证逻辑断裂、主张缺乏证据、内容与实验不一致
                │   C 补充推导/修正结论 → 回 D 重写 → 回 E 复审
                │
                └─ 实验问题 → C 调 B
                    实验不足以支撑主张、缺少关键消融/对比
                    B 补跑实验 → 回 C 更新结论 → 回 D 重写 → 回 E 复审
```

## 问题分级规则

E 层各审核工具的输出按以下规则分级：

| 回踢目标 | 触发条件 | 典型来源 |
|----------|---------|---------|
| **D 层** | 问题可通过重写/调整解决，不涉及研究内容本身 | E1 格式、E4 语言、E5 图表、E6 公式符号、E7 引用、E8 LLM痕迹 |
| **C 层** | 论证链有缺口，需要补充推导或修正研究结论 | E3 论证逻辑（断裂点）、E2 内容一致性（结论与证据矛盾） |
| **C→B** | C 层发现现有实验不足以填补缺口，需要新实验 | E3 触发 → C 评估 → 调 B0 设计实验 → B4–B6 执行 |

## 终止条件

- E 层所有审核工具全部通过 → 输出终稿
- 每轮回环 E 层输出的问题数量必须单调递减，若连续两轮未减少则人工介入

# D 层：论文写作

> D0 先搭好 LaTeX 骨架，D1–D5 并发写入各自的 .tex 文件，D6 汇编 main.tex 并在本地编译 PDF。
> 编译完成后触发 E2 一致性检查小回环——并发写作几乎必然产生前后不一致，先修到自洽再交 E 层全面审核。

- **输入**：C 层全部产出（推导森林、深层动机、贡献点、实验证据、文献关系图谱）
- **输出**：PDF 格式的完整论文

## D 层对外工具：附录填充

D 层向 DECB 超级回环暴露一个工具接口，供 E 层回踢修改时调用：

- **工具名**：`D.append_appendix(section_key, content)`
- **作用**：往附录对应章节中追加内容（补充实验细节、证明推导、超参数表等）
- **决策规则**：当 E 层触发论文修改时，优先判断：
  - **改正文**：核心论点、关键实验、叙事逻辑有问题 → 回踢 D1–D5 修改对应 .tex
  - **补附录**：细节不够但不影响主线（额外消融、完整证明、实现细节）→ 调此工具追加到附录

## 写作工具

### D0 模板初始化

在写作开始前搭好 LaTeX 项目骨架，确保后续 D1–D5 可以并发写入互不冲突。

- **输入**：目标 venue 名称
- **流程**：
  1. 调 **A4** 查找目标 venue 的官方 LaTeX 模板
  2. 在 git 仓库中创建论文目录，放入模板文件（.cls/.sty/.bst）
  3. 为每个章节创建空的 .tex 文件：`method.tex`、`experiments.tex`、`related_work.tex`、`introduction.tex`、`abstract.tex`、`conclusion.tex`、`appendix.tex`
  4. 创建占位 `main.tex`，用 `\input{}` 引入各章节文件
  5. 提交初始骨架到 git
- **输出**：可编译的空论文骨架（git 仓库中）

### D1 方法论写作

从 C 层的推导森林和深层动机中提取方法设计的逻辑链，写入 `method.tex`。

- **输入**：推导森林（主贡献分支）、深层动机
- **流程**：
  1. 从深层动机出发，构建"问题 → 洞察 → 设计"的叙事线
  2. 调 **A3** 确认方法描述与实际代码实现一致
  3. 调 **A4** 确认术语和符号与领域惯例一致
- **输出**：`method.tex`

### D2 实验章节写作

将 B 层的实验结果组织为 Experiment 章节，写入 `experiments.tex`。

- **输入**：实验证据（git 分支）、B 层性能表格和图表
- **流程**：
  1. 按贡献点组织实验：主实验 → 消融实验 → 分析实验
  2. 为每组实验撰写设置、结果、分析
  3. 确保每个实验结论都能回溯到 C 层的某个推导分支
- **输出**：`experiments.tex`

### D3 相关工作写作

从 C 层的文献关系图谱出发，写入 `related_work.tex`。

- **输入**：文献关系图谱
- **流程**：
  1. 从图谱中提取与本文相关的主要研究线
  2. 按研究线分组组织叙述，而非简单罗列论文
  3. 调 **A4** 补充遗漏的重要相关工作
  4. 明确本文与各研究线的区别和联系
- **输出**：`related_work.tex`

### D4 引言写作

写入 `introduction.tex`。

- **输入**：深层动机、主贡献、其他章节的已有内容
- **流程**：
  1. 构建引言叙事：领域背景 → 现存问题 → 本文洞察 → 方法概述 → 贡献总结
  2. 调 **A4** 确认背景描述和问题定位与领域现状一致
  3. 确保引言中的每个承诺在后文都有兑现
- **输出**：`introduction.tex`

### D5 摘要与结论写作

写入 `abstract.tex` 和 `conclusion.tex`。

- **输入**：全部已写章节
- **流程**：
  1. Abstract：用一段话概括问题、方法、关键结果
  2. Conclusion：总结贡献、讨论局限、展望未来方向
- **输出**：`abstract.tex` + `conclusion.tex`

### D6 论文编译

汇编 `main.tex` 并在本地编译为 PDF。

- **输入**：D0 的骨架 + D1–D5 写入的各 .tex 文件 + 图表 + 参考文献
- **流程**：
  1. 更新 `main.tex` 中的 `\input{}` 顺序、图表路径、参考文献
  2. 本地运行 `pdflatex` + `bibtex` 编译
  3. 编译成功 → 触发 **E2 一致性检查**（小回环）
- **输出**：PDF 论文

## D 层小回环：编译后一致性修复

```
D6 编译 PDF
    ↓
调 E2 文章一致性检查（仅 E2，不走完整 E 层）
    ↓
发现不一致（并发写作导致的术语/说法/数据矛盾）
    → 回踢到对应 D1–D5 修改 .tex → 重新 D6 编译 → 再调 E2
    ↓
E2 通过 → 交 E 层全面审核（进入 DECB 超级回环）
```

> 这个小回环只解决并发写作带来的表面不一致，不涉及研究内容层面的问题。
> 研究内容问题由 DECB 超级回环中 E 层的完整审核发现和回踢。

# E 层：论文审核

> 工具包——输入一份 PDF 格式的论文，从多个维度审核，输出审核报告。
> 审核不通过的问题按严重程度回踢至 D/C/B 层（见 DECB 超级回环）。

## E1 格式规范审核

检查论文是否符合目标投稿venue的格式要求。

- **输入**：PDF 论文、目标 venue（期刊/会议名）
- **流程**：
  1. 解析 PDF 的版面结构（页边距、字体、字号、单双栏、页数等）
  2. 调 **A4** 获取目标 venue 的格式模板要求
  3. 逐项比对，标记不符合项
- **输出**：格式审核报告（逐项通过/不通过 + 具体偏差）

## E2 内容一致性审核

检查论文内部的数据、图表、引用是否自洽。

- **输入**：PDF 论文
- **流程**：
  1. 提取正文中引用的所有数字、表格数据、图表结论
  2. 交叉比对：正文描述是否与图表实际内容一致、表格数据是否前后矛盾
  3. 检查引用完整性：正文中的引用是否都出现在参考文献中，参考文献是否有未被引用的条目
  4. 调 **A3** 比对论文中的代码/算法描述与实际代码是否一致
- **输出**：一致性审核报告（不一致项列表 + 具体位置）

## E3 论证逻辑审核

审核论文的论证链是否完整、是否有逻辑跳跃。

- **输入**：PDF 论文
- **流程**：
  1. 提取各章节的核心论点和支撑证据
  2. 检查：每个主张是否有对应的实验/理论支撑
  3. 检查：引言的 motivation → 方法的设计 → 实验的验证 是否形成闭环
  4. 标记逻辑断裂处（有主张无证据、有实验无结论、章节间衔接缺失）
  5. 调 **A4** 检索相关文献，验证论文中的关键主张是否与领域已有结论矛盾
- **输出**：逻辑审核报告（断裂点列表 + 修改建议）

## E4 语言质量审核

检查论文的语言表达质量。

- **输入**：PDF 论文
- **流程**：
  1. 检查语法、拼写、用词准确性
  2. 检查学术写作规范（被动语态使用、术语一致性、缩写首次出现是否展开等）
  3. 调 **A4** 检索目标 venue 的高引论文，对照其语言风格和表达习惯
  4. 标记表述模糊、冗余、或不够精确的段落
- **输出**：语言审核报告（问题列表 + 修改建议）

## E5 图表细节审核

逐张审核论文中的每个图和表。

- **输入**：PDF 论文
- **流程**：
  1. 逐张提取图表，对每张独立审核：
     - 图：是否清晰可读、坐标轴标注是否完整、图例是否齐全、配色是否可区分（含灰度打印场景）
     - 表：加粗/下划线标注的最优值是否真的是最优、列对齐和分隔是否清晰
  2. 检查每张图表的 caption 是否准确描述了其内容，是否能独立于正文被理解
  3. 调 **A4** 对照目标 venue 高引论文的图表风格
- **输出**：图表细节审核报告（逐张列出问题 + 修改建议）

## E6 公式与符号审核

逐个审核论文中的公式和数学符号。

- **输入**：PDF 论文
- **流程**：
  1. 提取所有公式及正文中出现的数学符号
  2. 检查符号一致性：同一含义是否全文使用同一符号、同一符号是否被赋予不同含义
  3. 检查每个符号是否在首次出现时有定义
  4. 检查公式编号与正文引用是否对得上
- **输出**：公式符号审核报告（不一致/未定义/引用错误列表）

## E7 引用细节审核

逐条审核参考文献的质量和规范性。

- **输入**：PDF 论文
- **流程**：
  1. 逐条检查引用信息完整性（作者、年份、venue、卷号/页码）
  2. 检查引用格式是否全文统一
  3. 调 **A4** 验证引用信息是否准确（论文是否存在、信息是否匹配）、是否有被撤回的论文
- **输出**：引用细节审核报告（问题条目列表 + 修正建议）

## E8 LLM 生成痕迹审核

检测并消除论文中的 LLM 生成痕迹，降低被识别为 AI 生成的风险。

- **输入**：PDF 论文
- **流程**：
  1. 检测典型 LLM 语言模式：过度使用的连接词（Furthermore, Moreover, It is worth noting...）、空洞的总结句、不自然的并列结构
  2. 检测内容层面的痕迹：过于面面俱到的叙述、缺乏个人学术判断的中立表述、不必要的免责声明
  3. 检测结构层面的痕迹：段落长度过于均匀、论述展开方式单一、缺乏领域特有的写作习惯
  4. 调 **A4** 检索目标领域的真人论文，对比写作风格差异
- **输出**：LLM 痕迹审核报告（疑似痕迹位置 + 具体模式 + 改写建议）

# 核心设计点

- **两个回环**：C 层内部的推导森林回环（深挖为什么有效）+ DECB 超级回环（写出来再审，审不过就回踢）。
- **A/F 层是全局工具包**，被 C/D/E 层按需调用。
- A4 是文献能力的统一入口，本地库作为缓存层不断积累，在线检索结果自动入库。
- 所有 C 层产物存 git，可追溯、可对比。
- **回环深度自适应**：E 层审核问题按严重程度分级，表层问题就地修，深层问题逐级回踢到 C 甚至 B。
